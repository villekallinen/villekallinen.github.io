GPU-Accelerated Climate Model Post-Processing

This project investigated how GPU computing can accelerate the post-processing of large atmospheric climate model datasets. The work focused on replacing computationally intensive CPU operations with CUDA-enabled GPU implementations using the CuPy library while maintaining numerical correctness.

The implementation was developed and evaluated using real climate simulation data on the CSC Mahti supercomputer.

Project Overview

Climate model simulations generate large multidimensional datasets that require substantial computational resources during post-processing. This project explored how GPU acceleration can reduce processing time by executing numerical operations in parallel using NVIDIA CUDA-enabled hardware.

The implementation was designed to preserve numerical correctness while reducing the execution time of computationally intensive post-processing operations.

GPU Processing Pipeline

  1. Load the NetCDF climate dataset.
  2. Select and transfer one time step to GPU memory.
  3. Execute the numerical calculations in parallel using CuPy.
  4. Transfer the calculated results back to the CPU.
  5. Release GPU memory that is no longer required.
  6. Repeat the process for all time steps in the dataset.

Performance Results

The GPU-accelerated implementation substantially reduced the total processing time compared with the serial CPU implementation. The largest improvement was achieved when the complete post-processing workflow was executed using the GPU.

In the performance comparison, the total processing time was reduced from approximately 461 seconds to 107 seconds. This corresponds to a speedup of approximately 4.3 times and a reduction of about 77% in execution time.

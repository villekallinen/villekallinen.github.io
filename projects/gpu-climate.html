GPU-Accelerated Climate Model Post-Processing

This project investigated how GPU computing can accelerate the post-processing of large atmospheric climate model datasets. The work focused on replacing computationally intensive CPU operations with CUDA-enabled GPU implementations using the CuPy library while maintaining numerical correctness. The implementation was developed and evaluated on the CSC Mahti supercomputer using real climate simulation data.

Project Overview

Climate model simulations generate extremely large datasets that require substantial computational resources during post-processing. This project explored how GPU acceleration can reduce processing time by executing numerical operations in parallel using NVIDIA CUDA-enabled hardware.

The implementation preserves numerical correctness while significantly reducing execution time for computationally intensive tasks.

GPU Processing Pipeline

  1. Load NetCDF climate datasets.
  2. Transfer one time step to GPU memory.
  3. Execute parallel computations using CuPy.
  4. Transfer results back to CPU.
  5. Release GPU memory.
  6. Repeat for the complete dataset.

Performance Results

The GPU-accelerated implementation reduced the total processing time significantly compared with the serial CPU implementation. The largest improvement was achieved when the full post-processing workflow was executed on the GPU.

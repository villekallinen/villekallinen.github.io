GPU-Accelerated Climate Model Post-Processing

This project focused on accelerating the post-processing of large climate model datasets by utilizing GPU computing. The objective was to replace computationally expensive CPU calculations with CUDA-enabled GPU implementations using CuPy while maintaining numerical correctness.

Project Goal

Global climate model simulations generate extremely large datasets whose post-processing can require several minutes or hours of computation. The goal of this project was to parallelize the existing CPU implementation and evaluate the performance benefits of GPU computing.

My Contribution

  • Implemented GPU versions of numerical algorithms using CuPy.
  • Converted NumPy-based computations into CUDA-compatible code.
  • Designed GPU memory management strategy.
  • Implemented GPU data transfer pipeline.
  • Performed performance benchmarking.
  • Validated GPU results against CPU implementation.

GPU Processing Pipeline

  1. Import NetCDF climate data.
  2. Transfer one time step to GPU memory.
  3. Execute CUDA kernels via CuPy.
  4. Transfer results back to CPU.
  5. Clear GPU memory.
  6. Repeat for all time steps.

Results

GPU computing significantly accelerated the complete post-processing workflow. The greatest performance gains were achieved when several atmospheric variables were computed simultaneously, whereas individual computations were limited by GPU memory transfer overhead.
